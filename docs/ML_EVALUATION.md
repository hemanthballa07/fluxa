# ML Fraud Scorer — Evaluation

_Generated by `ml/evaluate.py`. Model `v1`, ONNX-served._

## Scope & honesty notes

- **Distribution:** held-out **temporal** test split (newest 32,401 of 216,003 rows) of the
  **IEEE-CIS replay** stream. This is the surface where the model has signal.
- **Features are serve-parity only** (H1/H15): the request fields + transaction-time
  velocity/recency aggregates. The strong IEEE-CIS signal lives in the anonymized
  V/C/D blocks, which a live transaction cannot supply, so they are **deliberately excluded**
  to avoid train/serve skew. The headline number is therefore modest by design.
- **bankops sync traffic is feature-poor** (no categoricals, cold user) → the scorer is
  **advisory** there and rules stay primary.
- ONNX serving matches XGBoost to **3.4e-08** (no skew).

## ML vs rules (honest comparison)

The rules baseline here is **`amount > 500`** — the dominant, reproducible rule. It is a
weak, low-precision rule on IEEE-CIS (fraud is not amount-driven), so this bar is easy and
not the point; the point is that the ML curve dominates a single weak operating point.

| Approach | Precision | Recall |
|---|---|---|
| Rules-only (`amount>500`) | 0.0258 | 0.0276 |
| **ML-standalone @ rules' precision** | ~0.0258 | **1.0000** |
| Production blend (`rules OR ml>=0.87`) | 0.0800 | 0.1072 |

At the rules' precision level, the ML model reaches **1.0000** recall vs the rule's
**0.0276** — the lift comes from the categoricals + velocity aggregates, not from amount.

## Ablation — does ML beat amount alone?

The full model's 0.1320 vs amount-only 0.0431 confirms the lift is **not** just

## Reproducibility

- Training: `ml/train.py` (XGBoost, seed 42, temporal 70/15/15 split, `scale_pos_weight`).
- Threshold `tau=0.87` chosen on validation for FPR <= 1%.
- **The biggest quality lever is more data**: this model trained on 216,003 of
  ~590k available rows; with the full replay the velocity features stop being trivial. Re-run
  `make replay` to completion -> `make export-features` -> `make train` (zero code change).
